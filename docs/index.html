Anime Filler Skipper

Skip anime filler episodes automatically. Detects filler & mixed-canon episodes using animefillerlist.com and integrates with Jellyfin's media segment system.

https://raw.githubusercontent.com/mpash/jellyfin-anime-filler-skipper/main/manifest.json

Add this URL in Jellyfin → Dashboard → Plugins → Repositories

Installation

  1. Add repository Dashboard → Plugins → Repositories → paste the URL above → Save
  2. Install the plugin Go to Catalog → find "Anime Filler Skipper" → Install → restart Jellyfin
  3. Run the update task Dashboard → Scheduled Tasks → "Update Anime Filler Data" → run it now
  4. Enable segment skipping Configure the action for Unknown segments (see below)

How It Works

  1. Discover A daily scheduled task scans your libraries for anime series, maps each to its animefillerlist.com page, and fetches filler episode lists.
  2. Cache intelligently Data is cached locally with a 72-hour TTL. Subsequent fetches use ETag conditional requests — if the page hasn't changed, zero bytes are transferred.
  3. Mark as segment When you play an episode, the plugin checks if it's marked as filler. If so, a full-episode media segment (type: Unknown) is created — spanning the entire runtime.
  4. Skip with one click Jellyfin's segment system presents a "Skip" button. Skipping the segment jumps past the filler episode to the next one in your queue.

Features

Filler + Mixed Detection

Identifies both pure filler episodes and mixed canon/filler episodes. Configurable — toggle mixed treatment on or off.

ETag Cache Validation

Sends If-None-Match headers on every fetch. If animefillerlist.com returns 304 Not Modified, no data is re-downloaded.

Rate-Limit Respect

Configurable delay between requests (default 2s) and 72h cache TTL to avoid hammering animefillerlist.com.

Automatic Updates

Scheduled task refreshes filler data daily at 4:00 AM by default. Runs silently in the background.

Smart Slug Mapping

Automatically converts series names to animefillerlist.com slugs — "Attack on Titan" → "attack-on-titan".

Stale Cache Safety

If a fetch fails, the plugin keeps your existing cached data rather than discarding it — no gaps in filler detection.

Enable the Skip Button

Jellyfin's web client only auto-prompts for Intro and Outro segments by default. Filler episodes use segment type Unknown. You must configure the action for Unknown segments.

Run this API call against your Jellyfin server (replace the token):

curl -X POST "http://localhost:8096/DisplayPreferences/usersettings" \ -H "X-Emby-Authorization: YOUR_API_KEY" \ -H "Content-Type: application/json" \ -d '{"segmentTypeAction__Unknown":"AskToSkip"}'
Alternative: If your Jellyfin client has a Display Preferences UI for segment actions, set segmentTypeAction__Unknown to AskToSkip or Skip.

Configuration

After installing, go to Dashboard → Plugins → Anime Filler Skipper → Settings:

Cache Expiration (hours)

How long before re-fetching filler data. Default: 72. Filler lists rarely change — longer is more respectful to the upstream server.

Treat Mixed as Filler

When enabled (default), mixed canon/filler episodes are also marked as skippable. Disable to only skip pure filler.

Request Delay (ms)

Pause between HTTP requests when scanning multiple series. Default: 2000. Increase if the upstream server returns 429.
